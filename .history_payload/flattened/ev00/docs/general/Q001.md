# Q001: IPv4 Header Variable Length Parsing

**Status:** Open
**Date Opened:** Week 1 (v001)

## Question
The IPv4 header is theoretically variable length due to the "Options" field (though practically almost always 20 bytes, IHL=5). If a packet arrives with IP Options (IHL > 5), the byte offsets for the TCP/UDP ports will shift. 

Can a simple FSM-based parser safely ignore IP options, or must it dynamically shift its extraction offsets?

## Implications
If we use a fixed-offset FSM, any packet with IP options will result in garbage data extracted for the TCP/UDP ports.

2. Attempt to write a dynamic-offset FSM, but assess the impact on logic depth and 156.25 MHz timing closure.
