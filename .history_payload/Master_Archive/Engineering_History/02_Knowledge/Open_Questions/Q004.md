# Q004: Synaptic Weight Storage

**Status:** Open
**Date Opened:** Week 7 (v007)

## Question
The SNN requires Synaptic Weights for every connection. If we use a Fully Connected topology (8 inputs to 2 outputs), that's 16 weights. 

Should we store these weights in BRAM, or hardcode them into the RTL fabric (Distributed ROM)?

## Implications
If we use BRAM, we consume block memory resources and add 1 cycle of latency to the SNN computation.
If we hardcode them, the AI model cannot be updated without recompiling the FPGA bitstream.

## Decision Pending
For now (v007), we will hardcode the weights as `assign` statements to test the logic. This is logged as RTL Debt.
